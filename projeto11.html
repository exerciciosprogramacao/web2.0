Clicker

🎯 Objetivo do Projeto

- Clique na imagem para ganhar moedas.
- Use as moedas para comprar melhorias.
- Mais moedas por clique.
- Mais moedas por segundo.

👉 Um jogo simples onde o jogador evolui clicando e comprando upgrades.

📃 1. Selecionando os Elementos da Página

👉 Estas linhas de código usam document.querySelector( ) para obter referências a quatro elementos HTML essenciais para o jogo: imagem (o alvo do clique), contador (onde o total será exibida), contadorSegundos (onde o o valor por segundos será exibido) e cliques (quanto vale cada clique).

🖱 2. Mapeando os botões

👉 O código anexa um manipulador de eventos diretamente à propriedade .onclick do elemento HTML com o ID "btc1, btc2...". Quando este elemento for clicado, a função botaoc1, botaoc1... será executada.

🌎 3. Variáveis

👉 Estas linhas declaram e inicializam com let as variáveis que controlam o estado do jogo clicker: numeroCliques (o ganho por clique, começando em 1), numeroSegundos (ganho por segundo, começando em 0) e moedas (a pontuação total do jogador, começando em 0).

👉 Linhas 21, 22 e 23 usam a propriedade .innerText para inicializar a exibição no HTML da pontuação atual, valor por segundos e valor de cada clique.

🖼 4. Efeito da imagem

👉 O método .addEventListener() é usado para anexar um manipulador de eventos à variável imagem (a imagem principal do jogo). O evento que ele está escutando é o "click". Quando a imagem é clicada, a função anônima de callback será executada.

👉 O valor da variável moedas é incrementado pela quantidade armazenada em numeroCliques (moedas += numeroCliques é o mesmo que moedas = moedas + numeroCliques). Isso adiciona o ganho por clique à pontuação total.

👉 O conteúdo de texto (innerText) do elemento contador é atualizado para exibir o novo total de moedas ("Moedas: " + moedas).

👉 A propriedade de estilo CSS transform do elemento clicado (this, que se refere ao elemento imagem) é definida como "scale(95%)". Isso causa um efeito visual de encolhimento, simulando o "clique" na imagem.

👉 A função setTimeout( ) é chamada. Esta função nativa do JavaScript executa um bloco de código (a função de seta () => { ... }) apenas uma vez após um atraso especificado (o segundo argumento).

👉 O bloco de código dentro do setTimeout é definido. A propriedade de estilo CSS transform do elemento (this, ainda se referindo à imagem) é definida de volta para "scale(1)" (escala normal).

👉 O tempo de atraso (delay) do setTimeout é definido como 200 milissegundos. Após este breve período, o código na linha 32 será executado para restaurar a imagem ao seu tamanho normal.

⌚ 5. setInterval

👉 A função setInterval( ) é chamada. Esta função nativa do JavaScript executa repetidamente um bloco de código (a função anônima de callback) em intervalos de tempo fixos e contínuos (o segundo argumento). Este é o mecanismo do jogo para ganhos por segundo.

👉 O valor da variável moedas é incrementado pela quantidade armazenada em numeroSegundos. Como esta linha está dentro de um setInterval que roda a cada 1000ms (1 segundo), ela adiciona o ganho passivo à pontuação a cada segundo.

👉 O conteúdo de texto (innerText) do elemento contadorSegundos é atualizado. Ele exibe a string "Segundos: " concatenada com o valor de numeroSegundos, mostrando ao jogador o valor de ganho por segundo.

👉 O conteúdo de texto (innerText) do elemento contador é atualizado para exibir o novo total de moedas ("Moedas: " + moedas), refletindo o ganho passivo de moedas que acabou de ocorrer.

👉 O tempo de intervalo (interval) do setInterval é definido como 1000 milissegundos (ou seja, 1 segundo). Isso significa que as linhas 39 a 41 serão executadas a cada segundo.

💰 6. Upgrades

👉 O código define duas funções principais de melhoria, ambas chamadas por botões de compra. A função botaoc1 (ligada ao botão "btc1") gerencia a compra de um upgrade de clique: ela verifica se o jogador tem 50 moedas, subtrai o custo, incrementa a variável numeroCliques em 1, e atualiza o display de moedas e o display de ganho por clique. Já a função botaos1 (ligada ao botão "bts1") gerencia o upgrade de ganho passivo: ela verifica se o jogador tem 100 moedas, subtrai o custo, incrementa a variável numeroSegundos em 1 (aumentando o ganho por segundo), e atualiza o display de moedas e o display de ganho por segundo.

💰 7. Outros upgrades

👉 Você já entendeu como os botões dos upgrades funcionam. Tivemos o exemplo do botaoc1 (upgrade de clique) e do botaos1 (upgrade por segundo). Seu próximo passo é fazer o restante dos upgrades: botaoc2 até botaoc5 e botaos2 até botaos5. Lembre-se que cada função deve ter um custo crescente e, se o custo for pago, deve incrementar a variável de ganho correta (numeroCliques ou numeroSegundos) e atualizar os placares na tela. O nome da função deve sempre coincidir com o que foi atribuído ao .onclick no HTML.
