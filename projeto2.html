Simulação de Cadastro e Login

🎯 Objetivo

- Simular um cadastro simples, salvando Nome e Idade em elementos de exibição, uma "tabela".
- Implementar uma função de "login" que verifica se os dados digitados batem com os dados salvos.
- Focar na manipulação do conteúdo de elementos innerText e value.

👉 Este projeto é um exercício prático de leitura e escrita de dados no DOM, utilizando inputs e elementos de texto para armazenar e validar informações.

👉 Crie todo HTML e CSS, independente de como você for fazer, tenha em mente que você deve obter o mesmo resultado das imagens de exemplos.

👉 Usaremos 6 id ao todo, 1 em cada input, 1 para cada botão e 1 em cada uma das duas td.

🖱️ 1. Atribuindo Eventos aos Botões

👉 Novamente, usamos o método document.getElementById( ) para selecionar os botões de Cadastro e Verificação.

👉 As funções adicionar( ) e verificar( ) são atribuídas à propriedade .onclick para serem executadas quando o respectivo botão for clicado pelo usuário.

💾 2. Função adicionar( ) - Cadastrando os Dados

👉 As variáveis namep e agep recebem os dados digitados do formulário através da propriedade .value dos inputs.

👉 Em vez de criar novos elementos, usamos .innerText para sobrescrever o conteúdo de texto nos elementos de destino nometable e idadetable, salvando assim os dados cadastrados.

👉 A linha agep += " anos" usa concatenação para adicionar o texto " anos" junto com a idade antes de exibi-la. Ao final, os campos de input são limpos.

👉 No final, os campos de input são limpos, reatribuindo uma string vazia (" ") ao seu valor .value.

🔍 3. Função verificar( ) - Comparando as Informações

👉 Esta função primeiro lê dois conjuntos de dados. Primeiro os dados digitados nos inputs usando .value namep e agep. Segundo, os dados que foram previamente salvos na tabela, nametable e agetable, usando .innerText.

✅ 4. Lógica de Validação if/else

👉 A estrutura condicional if( ) é usada para tomar uma decisão. O operador lógico && (E) exige que as duas comparações sejam verdadeiras para que o Login seja efetuado. Ou seja: se o nome digitado for igual ao nome da tabela e a idade digitada for igual a idade da tabela.

👉 Se a condição for atendida, um alerta de sucesso é exibido e os campos de input são limpos, reatribuindo uma string vazia (" ") ao seu valor .value. Caso contrário, o bloco else é executado, exibindo uma mensagem de erro e os campos de input são limpos, reatribuindo uma string vazia (" ") ao seu valor .value.

👉 Quando adicionarmos alguém na tabela, ao lado da idade da pessoa, deve aparacer escrito: anos, assim como na imagem abaixo.

👉 Após o cadastro, quando fizermos o login lembre-se que na idade do input não podemos escrever anos, só números e que na tabela vai constar a idade com a escrita de anos.
