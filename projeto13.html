Git/GitHub

🎯 Objetivo do Projeto

- Aprender sobre o sistema de controle de versão que permite registrar e acompanhar as mudanças feitas em um projeto, especialmente em códigos de programação. 
- Voltar a versões anteriores, trabalhar em equipe e evitar perda de dados.

📺 Entendendo os conceitos

👉 Este video contem toda parte explicativa sobre como foi criado o git seu uso e pra que ele serve. Logo abaixo teremos a explicação pratica dos comandos usados.

🗃 1. Criar repositório no GitHub

👉 Primeiro, acesse github.com e faça login. Clique no botão Repositories.

👉 Clique em "New" para criar um repositório.

👉 Só precisamos dar um nome, clique em Repository name *, deixe o restante como está.

🗃 2. Clonar repositório no VS Code

  git clone https://github.com/SEU_USUARIO/NOME_DO_REPOSITORIO.git

👉 Copie a URL do repositório criado e use o comando git clone no terminal do VS Code. Isso cria uma cópia local do repositório no seu computador.

🗃 3. Entrar na pasta do projeto

  cd NOME_DO_REPOSITORIO

👉 Use o comando cd para entrar na pasta do projeto que você acabou de clonar. Todos os comandos Git seguintes devem ser executados dentro dessa pasta.

🗃 4. Verificar status do repositório

  git status

👉 O comando git status mostra os arquivos que foram modificados, os que estão prontos para commit e se há algo não rastreado pelo Git.

🗃 5. Adicionar arquivos para commit

  git add .

👉 O git add . adiciona todos os arquivos modificados ou novos à "área de stage", preparando-os para serem salvos no próximo commit.

🗃 6. Criar um commit

  git commit -m "Mensagem descrevendo as alterações"

👉 O git commit cria um ponto de salvamento no histórico do Git. A mensagem (-m) deve resumir as alterações feitas.

🗃 7. Enviar alterações para o GitHub

  git push origin main

👉 O git push envia os commits locais para o repositório remoto no GitHub. Certifique-se de substituir "main" pelo nome correto da sua branch principal, se for diferente.

🗃 8. Atualizar repositório local com alterações remotas

  git pull origin main

👉 Use git pull para trazer alterações do repositório remoto para o seu repositório local. Isso é importante se outras pessoas estiverem trabalhando no mesmo projeto.
